# 📌 1. Advanced Validation (Deep Dive)

FastAPI uses **Pydantic** for validation. It ensures that incoming data is correct before your business logic runs.

👉 Why it matters:
- Prevents invalid data from entering your system
- Reduces bugs
- Makes APIs self-documented

---

## 🔤 String Validation

```python
from pydantic import BaseModel, Field

class User(BaseModel):
    username: str = Field(
        ...,
        min_length=3,
        max_length=20,
        regex="^[a-zA-Z0-9_]+$"
    )
```
💡 Explanation:

... → required field

min_length / max_length → restrict size

regex → enforce format (only letters, numbers, underscore)


👉 Real-world use:

Username validation

Product codes

Slugs



---

🔢 Numeric Validation

class Product(BaseModel):
    price: float = Field(gt=0, lt=100000)
    quantity: int = Field(ge=1, le=100)

💡 Explanation:

gt → greater than

ge → greater than or equal

lt → less than

le → less than or equal


👉 Real-world use:

Price must be positive

Quantity limits in inventory systems



---

📧 Built-in Types (Powerful 💡)

class User(BaseModel):

💡 Explanation:

EmailStr → validates email format

HttpUrl → validates URLs


👉 Real-world use:

Signup forms

🎯 Custom Validation (Field Validator)

from pydantic import validator

class User(BaseModel):
    password: str

    @validator("password")
    def strong_password(cls, value):
        if len(value) < 8:
            raise ValueError("Password must be at least 8 characters")
        return value

💡 Explanation:

Custom logic for validation

Runs automatically before API execution


👉 Real-world use:

Password strength rules

Business-specific validations



---

🔥 Multiple Field Validation (Root Validator)

from pydantic import root_validator

class Order(BaseModel):
    price: float
    discount: float

    @root_validator
    def check_discount(cls, values):
        if values["discount"] > values["price"]:
            raise ValueError("Discount cannot exceed price")
        return values

💡 Explanation:

Validates multiple fields together

Used for cross-field logic


👉 Real-world use:

Discount vs price

Start date vs end date



---

📌 2. Dependency Injection with Depends()

FastAPI has built-in Dependency Injection (DI) — similar to .NET Core.

👉 Why it matters:

Reusable logic

Cleaner code

Separation of concerns



---

✅ Basic Dependency

from fastapi import Depends

def common_query_params(q: str = None):
    return {"q": q}

@app.get("/items/")
def read_items(params: dict = Depends(common_query_params)):
    return params

💡 Explanation:

Depends() injects function result

Avoids repeating logic



---

🔄 Database Dependency (Important)

def get_db():
    db = "DB Connection"
    try:
        yield db
    finally:
        print("Closing DB")

@app.get("/users/")
def get_users(db=Depends(get_db)):
    return {"db": db}

💡 Explanation:

yield allows setup + cleanup

Similar to using in C#


👉 Real-world use:

DB connections

Transactions



---

🎯 Dependency with Parameters

def pagination(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

@app.get("/items/")
def get_items(p=Depends(pagination)):
    return p

💡 Explanation:

Automatically reads query params

Injects into function



---

🔐 Authentication Dependency

def get_current_user(token: str):
    if token != "secret":
        raise Exception("Unauthorized")
    return {"user": "admin"}

@app.get("/secure/")
def secure_route(user=Depends(get_current_user)):
    return user

💡 Explanation:

Centralized auth logic

Reusable across endpoints

📌 3. Advanced Dependencies (Real-World Patterns)


---

🔥 Class-Based Dependency

class CommonParams:
    def __init__(self, q: str = None, limit: int = 10):
        self.q = q
        self.limit = limit

@app.get("/items/")
def get_items(params: CommonParams = Depends()):
    return params.__dict__

💡 Explanation:

Cleaner for complex dependencies

Groups related parameters



---

🔁 Reusable Dependency

def verify_api_key(api_key: str):
    if api_key != "123":
        raise Exception("Invalid API Key")

@app.get("/data/")
def get_data(dep=Depends(verify_api_key)):
    return {"data": "secure"}


---

🔄 Dependency Chain (Nested DI)

def get_token():
    return "token123"

def get_user(token=Depends(get_token)):
    return {"user": token}

@app.get("/profile/")
def profile(user=Depends(get_user)):
    return user

💡 Explanation:

Dependencies can depend on other dependencies

Builds layered architecture



---

🌍 Global Dependencies

app = FastAPI(dependencies=[Depends(verify_api_key)])

💡 Explanation:

Applied to all routes

Useful for auth/logging



---

⚡ Dependency with Yield

def get_resource():
    print("Start")
    yield "resource"
    print("Cleanup")

@app.get("/")
def use_resource(res=Depends(get_resource)):
    return {"res": res}

💡 Explanation:

Setup before request

Cleanup after response

📌 4. JSON Compatible Encoder


---

🧠 Problem

Python objects like datetime are not JSON serializable.


---

✅ Solution

from fastapi.encoders import jsonable_encoder
from datetime import datetime

data = {
    "time": datetime.now()
}

json_data = jsonable_encoder(data)


---

💡 Explanation

jsonable_encoder converts:

datetime → string

Pydantic model → dict

complex objects → JSON safe



---

🎯 Example with API

@app.post("/items/")
def create_item(item: Item):
    json_item = jsonable_encoder(item)
    return json_item


---

🔄 Nested Conversion

class Item(BaseModel):
    name: str
    created_at: datetime

@app.post("/items/")
def create(item: Item):
    return jsonable_encoder(item)


---

⚠️ When to Use

Before saving to DB

Before sending response

When handling datetime
