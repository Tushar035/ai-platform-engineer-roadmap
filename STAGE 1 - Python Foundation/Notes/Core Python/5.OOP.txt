🐍 Object-Oriented Programming (OOP) in Python is a programming paradigm based on objects and classes. It helps in writing modular, reusable, scalable, and maintainable code.

Since you are a .NET developer, you will find Python OOP concepts very similar to C# classes and objects, but Python is more flexible and dynamic.

📌 1. Class and Object
-------------------
Class
A class is a blueprint for creating objects.

Example:
A class Car defines properties like brand, color, speed and behaviors like drive, brake.

Object
An object is an instance of a class.

Example:
class Car:
    brand = "Toyota"
    color = "Red"

car1 = Car()
print(car1.brand)
print(car1.color)

Output
Toyota
Red

Here:
Car → class
car1 → object

📌 2. Constructor (__init__)
---------------------------
A constructor is used to initialize object properties.

In Python the constructor is:
__init__()

It runs automatically when an object is created.
Example
class Car:
    def __init__(self, brand, color):
        self.brand = brand
        self.color = color

car1 = Car("BMW", "Black")

print(car1.brand)
print(car1.color)

Output
BMW
Black

Explanation:
Keyword	Meaning

__init__	constructor
self	reference to current object
self.brand	object variable

📌 3. Instance Variables vs Class Variables
---------------------------------------
Class Variable:
 Shared by all objects.
Instance Variable:
 Unique for each object.

Example
class Employee:
    company = "Microsoft"   # class variable

    def __init__(self, name, salary):
        self.name = name
        self.salary = salary  # instance variable

emp1 = Employee("Tushar", 50000)
emp2 = Employee("Rahul", 60000)

print(emp1.company)
print(emp2.company)

Output
Microsoft
Microsoft

📌 4. Methods in Python Classes
-------------------------------
A method is a function inside a class.

Types:
1. Instance Method
2. Class Method
3. Static Method


4.1 Instance Method
--------------------
Works with object data.
class Student:

    def __init__(self, name):
        self.name = name

    def greet(self):
        print("Hello", self.name)

s1 = Student("Tushar")
s1.greet()

Output
Hello Tushar

4.2 Class Method
------------------
Works with class variables.
Decorator:
@classmethod
Example
class Employee:
    company = "Google"

    @classmethod
    def change_company(cls, new_name):
        cls.company = new_name

Employee.change_company("Microsoft")
print(Employee.company)
Output
Microsoft

4.3 Static Method
-----------------
Does not use class or instance.

Decorator:
@staticmethod

Example
class MathUtils:

    @staticmethod
    def add(a, b):
        return a + b

print(MathUtils.add(5, 3))
Output
8

📌 5. Encapsulation
--------------------
Encapsulation means restricting direct access to variables and protecting data.

Python uses:
| Type      | Syntax       | Where Accessible                                |                                                                                                                                                                                                                                                   |
| --------- | ------------ | ------------------------------------------------|
| Public    | variable     | Anywhere (inside class, outside class, subclass)| 
| Protected | _variable    | Inside class + subclass (by convention only)    |
| Private   | __variable   | Only inside the class (name mangling applied)   |  

Example
class BankAccount:
    def __init__(self, balance):
        self.__balance = balance   # private variable

    def deposit(self, amount):
        self.__balance += amount

    def get_balance(self):
        return self.__balance

account = BankAccount(1000)
account.deposit(500)
print(account.get_balance())

Output
1500

Direct access not allowed:
account.__balance ❌

💡 Important Insight (Interview Level)

Python does not have true private variables like C# or Java.
It uses name mangling to avoid accidental access, not strict security.

# Protected Example
class Employee:
    def __init__(self, name, salary):
        self.name = name          # public
        self._salary = salary     # protected

class Manager(Employee):
    def display(self):
        # ✅ accessing protected variable from parent class
        print(f"Manager Salary: {self._salary}")

# Create objects
emp = Employee("Tushar", 50000)
mgr = Manager("Rahul", 80000)

# Access inside subclass
mgr.display()   # ✅ allowed

# Access outside class
print(emp._salary)  # ⚠️ allowed but NOT recommended

📌 6. Inheritance
----------------
Inheritance allows one class to reuse another class's properties and methods.

Parent class
     ↓
Child class
Example
class Animal:
    def speak(self):
        print("Animal makes sound")

class Dog(Animal):
    def bark(self):
        print("Dog barks")

d = Dog()
d.speak()
d.bark()

Output
Animal makes sound
Dog barks

Types of Inheritance
## 1️⃣ Single Inheritance
👉 One parent → One child

python
class Parent:
    def show(self):
        print("This is Parent")

class Child(Parent):
    def display(self):
        print("This is Child")

c = Child()
c.show()     # inherited
c.display()

## 2️⃣ Multiple Inheritance
👉 One child → Multiple parents

python
class Father:
    def skill1(self):
        print("Driving")

class Mother:
    def skill2(self):
        print("Cooking")

class Child(Father, Mother):
    def skills(self):
        print("Child skills")

c = Child()
c.skill1()   # from Father
c.skill2()   # from Mother

## 3️⃣ Multilevel Inheritance
👉 Grandparent → Parent → Child

python
class Grandparent:
    def home(self):
        print("Grandparent's Home")

class Parent(Grandparent):
    def car(self):
        print("Parent's Car")

class Child(Parent):
    def bike(self):
        print("Child's Bike")

c = Child()
c.home()  # from Grandparent
c.car()   # from Parent
c.bike()

## 4️⃣ Hierarchical Inheritance
👉 One parent → Multiple children

python
class Parent:
    def property(self):
        print("Parent Property")

class Child1(Parent):
    def skill1(self):
        print("Child1 Skill")

class Child2(Parent):
    def skill2(self):
        print("Child2 Skill")

c1 = Child1()
c2 = Child2()

c1.property()
c2.property()

## 5️⃣ Hybrid Inheritance

👉 Combination of multiple types (e.g., multiple + multilevel)

python
class A:
    def methodA(self):
        print("Class A")

class B(A):
    def methodB(self):
        print("Class B")

class C(A):
    def methodC(self):
        print("Class C")

class D(B, C):
    def methodD(self):
        print("Class D")

d = D()
d.methodA()  # from A
d.methodB()  # from B
d.methodC()  # from C
d.methodD()

# 🔥 Quick Summary

| Type         | Structure    |
| ------------ | ------------ |
| Single       | A → B        |
| Multiple     | A + B → C    |
| Multilevel   | A → B → C    |
| Hierarchical | A → B, A → C |
| Hybrid       | Combination  |

## ⚡ Pro Tip (Very Important)
* Python uses **MRO (Method Resolution Order)** to resolve conflicts in multiple/hybrid inheritance.
* You can check it using:

python
print(ClassName.mro())

# 💎 Diamond Problem (Visual)
### Structure:

        A
       / \
      B   C
       \ /
        D

👉 Class **D** inherits from both **B** and **C**,
and both B & C inherit from **A**

# ⚠️ The Problem
👉 If method exists in multiple places, **which one should Python call?**

## 🧪 Example
python
class A:
    def show(self):
        print("A")

class B(A):
    def show(self):
        print("B")

class C(A):
    def show(self):
        print("C")

class D(B, C):
    pass

d = D()
d.show()

## 🤔 What will be output?
👉 Output:
B

# 🧠 Why "B"? → MRO comes into play
Python follows a specific order called:

# 🔥 MRO (Method Resolution Order)
👉 It decides **which class to search first**

## 📌 Check MRO
python
print(D.mro())

👉 Output:
[D, B, C, A, object]


## 🧭 How Python searches:

1. D
2. B ✅ (found `show()`)
3. C
4. A

👉 So it stops at **B**

# ⚙️ Rule Behind MRO (C3 Linearization)
👉 Python uses **C3 Linearization algorithm**

### Rules:
* Left → Right priority
* Child comes before parents
* No duplicate paths
* Maintains hierarchy order

# 🔥 Important Twist (Using super())
python
class A:
    def show(self):
        print("A")

class B(A):
    def show(self):
        print("B")
        super().show()

class C(A):
    def show(self):
        print("C")
        super().show()

class D(B, C):
    def show(self):
        print("D")
        super().show()

d = D()
d.show()

## 🎯 Output:
D
B
C
A

## 🧠 Why this order?
👉 Because MRO is:

[D, B, C, A, object]
👉 `super()` follows **this exact order**, not just parent class!

# 🚀 Real Insight (Very Important)
> `super()` does NOT mean "call parent"
> It means → **call next class in MRO**

# ⚡ Interview Summary
* Diamond problem = ambiguity in multiple inheritance
* Python solves it using **MRO (C3 Linearization)**
* `super()` follows MRO, not direct parent
* Order = **Left → Right → Depth-aware**


📌 7. Polymorphism
----------------
Polymorphism means same method name but different behavior.

Example
class Dog:
    def sound(self):
        print("Dog barks")

class Cat:
    def sound(self):
        print("Cat meows")


animals = [Dog(), Cat()]

for animal in animals:
    animal.sound()

Output
Dog barks
Cat meows

📌 8. Method Overriding
----------------------
Child class overrides parent method.

Example
class Animal:
    def speak(self):
        print("Animal sound")


class Dog(Animal):
    def speak(self):
        print("Dog barks")

d = Dog()
d.speak()

Output
Dog barks

📌 9. Abstraction
----------------
Abstraction hides internal implementation and only shows essential features.

In Python we use:
abc module

Example
from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def start(self):
        pass

class Car(Vehicle):
    def start(self):
        print("Car started")

c = Car()
c.start()

Output
Car started

📌 10. Magic Methods (Dunder Methods)
------------------------------------
Python special methods start with double underscore.

Examples
Method	Purpose
__init__	constructor
__str__	string representation
__len__	length
__add__	addition
__eq__	equality

Example
class Person:
    def __init__(self, name):
        self.name = name

    def __str__(self):
        return f"Person name is {self.name}"

p = Person("Tushar")
print(p)

Output
Person name is Tushar

📌 11. Real-world Example (Important)
----------------------------------
Example: E-commerce Order System

class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

class Order:
    def __init__(self):
        self.products = []

    def add_product(self, product):
        self.products.append(product)

    def total_price(self):
        return sum(p.price for p in self.products)


p1 = Product("Laptop", 80000)
p2 = Product("Mouse", 500)

order = Order()

order.add_product(p1)
order.add_product(p2)

print(order.total_price())
Output
80500
