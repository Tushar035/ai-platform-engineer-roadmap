🧠 2. Iterators & Generators
----------------------------
#🔁 Iterators
--------------
An iterator is an object that you can loop over.

Example:
python
nums = [1, 2, 3]
it = iter(nums)

print(next(it))  1
print(next(it))  2

#⚡ Generators
----------------
Generators are memory-efficient iterators.

python
def count_up(n):
    i = 1
    while i <= n:
        yield i
        i += 1

gen = count_up(5)

for num in gen:
    print(num)

👉 Uses `yield` instead of `return`

🔥 Why Generators Matter
--------------------------
* Used in large datasets
* Used in AI pipelines
* Saves memory (lazy loading)

🧵 3. Decorators
----------------
Decorators allow you to modify function behavior without changing code.

#Example:
python
def my_decorator(func):
    def wrapper():
        print("Before function")
        func()
        print("After function")
    return wrapper

@my_decorator
def say_hello():
    print("Hello")

say_hello()

Output:
Before function
Hello
After function

#Real Use Case
--------------
* Logging
* Authentication (FastAPI)
* Performance tracking

🧩 4. Lambda, Map, Filter, Reduce
----------------------------------
#Lambda
---------
python
square = lambda x: x*x
print(square(5))

#Map
-----
python
nums = [1,2,3]
result = list(map(lambda x: x*2, nums))

#Filter
-------
python
nums = [1,2,3,4]
even = list(filter(lambda x: x%2==0, nums))

#Reduce
--------
python
from functools import reduce

nums = [1,2,3,4]
result = reduce(lambda x,y: x+y, nums)

🔒 6. Closures
---------------
A closure remembers variables from outer scope.

python
def outer(x):
    def inner(y):
        return x + y
    return inner

add5 = outer(5)
print(add5(3))  8

🧱 7. Context Managers (`with` statement)
--------------------------------------
Used for resource management.

python
with open("file.txt", "r") as f:
    data = f.read()

👉 Automatically closes file

#Custom Context Manager
----------------------
python
class MyContext:
    def __enter__(self):
        print("Enter")
    def __exit__(self, exc_type, exc_val, exc_tb):
        print("Exit")

with MyContext():
    print("Inside")


🧵 8. Multithreading vs Multiprocessing
-----------------------------------
#Threading
-----------
python
import threading

def task():
    print("Task running")

t = threading.Thread(target=task)
t.start()

#Multiprocessing
----------------
python
from multiprocessing import Process

def task():
    print("Process running")

p = Process(target=task)
p.start()

#Difference
-------------
| Feature | Threading    | Multiprocessing |
| ------- | ------------ | --------------- |
| Memory  | Shared       | Separate        |
| Speed   | Good for I/O | Good for CPU    |
| GIL     | Affected     | Not affected    |

⚡ 9. Async Programming (IMPORTANT)
--------------------------------
Used in:
* FastAPI
* APIs
* High-performance systems

#Example:
python
import asyncio

async def task():
    print("Start")
    await asyncio.sleep(2)
    print("End")

asyncio.run(task())

🧠 10. GIL (Global Interpreter Lock)
-------------------------------------
* Only one thread executes Python bytecode at a time
* Limits CPU-bound threading

👉 Solution:
* Use multiprocessing
* Use async

🧬 13. Metaclasses (Advanced Topic)
-----------------------------------
Metaclass controls class creation.

python
class Meta(type):
    def __new__(cls, name, bases, dct):
        print("Creating class")
        return super().__new__(cls, name, bases, dct)

class MyClass(metaclass=Meta):
    pass
