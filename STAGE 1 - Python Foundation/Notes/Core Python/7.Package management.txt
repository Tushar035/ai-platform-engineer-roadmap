# ❌ 2. Problem Without Package Management
-------------------------------------------
Imagine you build a project using:

text
fastapi==0.110
numpy==1.26


Now your teammate installs:

text
fastapi==0.95
numpy==1.20

💥 Your project may break!


# ✅ 3. Solution
-----------------
Use package managers like:
* pip → default Python tool
* poetry → modern, advanced tool

# 🔹 PART 1 — pip (Basic & Most Used)
-------------------------------------
# ⚙️ 4. What is pip?
-------------------
`pip` is Python’s default package manager.
👉 It comes pre-installed with Python.
Always start with: Directory

Because:
pip and poetry manage projects (folders), not just single files
You need a project structure to practice properly
For pip Practice (Traditional Way)
📁 Steps
Click Directory
Name it: pip_project
Inside it → create a Python File → main.py

Then in terminal:
python -m venv venv
venv\Scripts\activate   # Windows

# 📥 5. Installing Packages
-----------------------------
bash
pip install fastapi

Install multiple:
bash
pip install fastapi uvicorn

# 🔍 6. Check Installed Packages
----------------------------------
bash
pip list

# 📌 7. Install Specific Version
--------------------------------
bash
pip install fastapi==0.110.0


# 🔄 8. Upgrade Package
-------------------------
bash
pip install --upgrade fastapi

# ❌ 9. Uninstall Package
---------------------------
bash
pip uninstall fastapi

# 📄 10. requirements.txt (VERY IMPORTANT)
------------------------------------------
Save dependencies:

bash
pip freeze > requirements.txt

Example:
---------
text
fastapi==0.110.0
uvicorn==0.29.0

## Install from file
----------------------
bash
pip install -r requirements.txt

👉 Used in:
* Team collaboration
* Deployment
* CI/CD pipelines

# ⚠️ 11. Problems with pip
---------------------------
Even though pip is simple, it has limitations:
❌ No automatic dependency resolution (older versions)
❌ No project-level configuration
❌ No lock file (exact reproducibility issue)*
❌ Manual version control

# 🚀 PART 2 — poetry (Modern Tool)
-----------------------------------
# ⚙️ 12. What is poetry?
------------------------
poetry is an advanced dependency manager and packaging tool.

👉 It replaces:
* pip
* requirements.txt
* virtualenv (partially)

# 💡 Why Use Poetry?
✔ Automatic dependency resolution
✔ Built-in virtual environment
✔ Lock file for exact versions
✔ Cleaner project structure

For poetry Practice (Modern Way)

👉 DO NOT manually create files first

📁 Steps
Create a Directory → poetry_project
Open terminal inside it
Then run:
poetry init
# OR (better)
poetry new my_app

# 📥 13. Install Poetry
-----------------------
bash
pip install poetry

Verify:

bash
poetry --version

# 🛠️ 14. Create New Project
-------------------------------
bash
poetry new my_project
cd my_project

# 📦 15. Add Dependencies

-------------------------------------

toml

---------------------------
## poetry.lock

# 📊 21. pip vs poetry (Important Comparison)
----------------------------------------------
| Feature               | pip       | poetry          |
| --------------------- | --------- | --------------- |
| Ease of use           | Easy      | Moderate        |
| Dependency resolution | Basic     | Advanced        |
| Lock file             | ❌         | ✅               |
| Virtual env           | Manual    | Built-in        |
| Best for              | Beginners | Production apps |

# 🧠 22. When to Use What?
----------------------------
### Use pip when:
✔ Learning Python
✔ Small scripts
✔ Simple projects

### Use poetry when:
✔ Building APIs (FastAPI)
✔ AI/ML projects
✔ Production systems
✔ Team collaboration

# ⚠️ 25. Common Mistakes
-------------------------
❌ Installing packages globally
❌ Not using version pinning
❌ Ignoring requirements.txt
❌ Mixing pip and poetry incorrectly

# 🧩 26. Best Practices
----------------------------
✔ Always use virtual environment
✔ Pin versions (fastapi==x.x.x)
✔ Use lock files in production
✔ Keep dependencies minimal
✔ Regularly update packages

# 🚀 27. Advanced Concepts (Industry Level)
-------------------------------------------

bash
pipdeptree

## Dev Dependencies (poetry)
bash

## Export requirements from poetry
bash

# 🎯 28. Interview Questions
-------------------------------
### Q1: What is pip?
👉 Python package manager used to install libraries

### Q2: What is poetry?
👉 Advanced dependency and package manager with lock file support

### Q3: Difference between requirements.txt and poetry.lock?

| File             | Purpose                         |
| ---------------- | ------------------------------- |
| requirements.txt | Manual dependency list          |
| poetry.lock      | Exact versions (auto generated) |
