# 📦 Python Package Management — Complete Beginner Guide (pip & poetry)
--------------------------------------------------------------------------
If you're learning Python for **backend, FastAPI, AI/ML, or DevOps**, understanding **package management** is essential.

Because in real projects, you don’t just write code — you manage **dependencies, versions, environments, and reproducibility**.

Let’s go step by step 👇

---

# 🧠 1. What is Package Management?

A **package** is a collection of reusable Python code.

Examples:

* `fastapi` → for APIs
* `numpy` → for numerical computing
* `pandas` → for data analysis

👉 **Package management** means:

* Installing packages
* Updating them
* Managing versions
* Removing unused ones

---

# ❌ 2. Problem Without Package Management

Imagine you build a project using:

```text
fastapi==0.110
numpy==1.26
```

Now your teammate installs:

```text
fastapi==0.95
numpy==1.20
```

💥 Your project may break!

---

# ✅ 3. Solution

Use package managers like:

* **pip** → default Python tool
* **poetry** → modern, advanced tool

---

# 🔹 PART 1 — pip (Basic & Most Used)

---

# ⚙️ 4. What is pip?

`pip` is Python’s default package manager.

👉 It comes pre-installed with Python.

---

# 📥 5. Installing Packages

```bash
pip install fastapi
```

Install multiple:

```bash
pip install fastapi uvicorn
```

---

# 🔍 6. Check Installed Packages

```bash
pip list
```

---

# 📌 7. Install Specific Version

```bash
pip install fastapi==0.110.0
```

---

# 🔄 8. Upgrade Package

```bash
pip install --upgrade fastapi
```

---

# ❌ 9. Uninstall Package

```bash
pip uninstall fastapi
```

---

# 📄 10. requirements.txt (VERY IMPORTANT)

Save dependencies:

```bash
pip freeze > requirements.txt
```

Example:

```text
fastapi==0.110.0
uvicorn==0.29.0
```

---

## Install from file

```bash
pip install -r requirements.txt
```

👉 Used in:

* Team collaboration
* Deployment
* CI/CD pipelines

---

# ⚠️ 11. Problems with pip

Even though pip is simple, it has limitations:

❌ No automatic dependency resolution (older versions)
❌ No project-level configuration
❌ No lock file (exact reproducibility issue)
❌ Manual version control

---

# 🚀 PART 2 — poetry (Modern Tool)

---

# ⚙️ 12. What is poetry?

**poetry** is an advanced dependency manager and packaging tool.

👉 It replaces:

* pip
* requirements.txt
* virtualenv (partially)

---

# 💡 Why Use Poetry?

✔ Automatic dependency resolution
✔ Built-in virtual environment
✔ Lock file for exact versions
✔ Cleaner project structure

---

# 📥 13. Install Poetry

```bash
pip install poetry
```

Verify:

```bash
poetry --version
```

---

# 🛠️ 14. Create New Project

```bash
poetry new my_project
cd my_project
```

---

# 📦 15. Add Dependencies

```bash
poetry add fastapi
```

👉 Automatically:

* installs package
* updates config
* updates lock file

---

# 📄 16. Important Files in Poetry

---

## pyproject.toml

```toml
[tool.poetry.dependencies]
python = "^3.11"
fastapi = "^0.110"
```

👉 Main configuration file

---

## poetry.lock

👉 Stores exact versions

✔ Ensures same environment everywhere

---

# ▶️ 17. Run Project

```bash
poetry run python main.py
```

---

# 🧪 18. Activate Shell

```bash
poetry shell
```

---

# 🔄 19. Update Dependencies

```bash
poetry update
```

---

# ❌ 20. Remove Package

```bash
poetry remove fastapi
```

---

# 📊 21. pip vs poetry (Important Comparison)

| Feature               | pip       | poetry          |
| --------------------- | --------- | --------------- |
| Ease of use           | Easy      | Moderate        |
| Dependency resolution | Basic     | Advanced        |
| Lock file             | ❌         | ✅               |
| Virtual env           | Manual    | Built-in        |
| Best for              | Beginners | Production apps |

---

# 🧠 22. When to Use What?

### Use pip when:

✔ Learning Python
✔ Small scripts
✔ Simple projects

---

### Use poetry when:

✔ Building APIs (FastAPI)
✔ AI/ML projects
✔ Production systems
✔ Team collaboration

---

---

## Using poetry

# 📁 24. Project Structure Comparison

---

## pip

```text
project/
├── venv/
├── main.py
├── requirements.txt
```

---

## poetry

```text
project/
├── pyproject.toml
├── poetry.lock
├── main.py
```

---

# ⚠️ 25. Common Mistakes

❌ Installing packages globally
❌ Not using version pinning
❌ Ignoring requirements.txt
❌ Mixing pip and poetry incorrectly

---

# 🧩 26. Best Practices

✔ Always use virtual environment
✔ Pin versions (fastapi==x.x.x)
✔ Use lock files in production
✔ Keep dependencies minimal
✔ Regularly update packages

---

# 🚀 27. Advanced Concepts (Industry Level)

---

## Dependency Tree

```bash
pipdeptree
```

---

## Dev Dependencies (poetry)

```bash
poetry add pytest --group dev
```

---

## Export requirements from poetry

```bash
poetry export -f requirements.txt --output requirements.txt
```

---

### Q2: What is poetry?

### Q3: Difference between requirements.txt and poetry.lock?

| File             | Purpose                         |
| ---------------- | ------------------------------- |
| requirements.txt | Manual dependency list          |
| poetry.lock      | Exact versions (auto generated) |
